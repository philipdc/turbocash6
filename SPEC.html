Overview

TurboCASH 6 is a modern, open-source accounting and financial management web application built with modern web technologies. This specification document outlines the structure, features, and technical architecture of the application.

License: MIT License - Open for contributions

Purpose: Provide a comprehensive, user-friendly accounting solution accessible via web browser

Core Features

1. Dashboard

  • Financial overview with real-time statistics
  • Recent transactions display
  • Quick action buttons
  • Customizable widgets

2. Accounts Management

  • Chart of accounts
  • Account creation and management
  • Account balances and history
  • Multi-currency support

3. Transaction Processing

  • Journal entries
  • Transaction recording
  • Batch processing
  • Transaction search and filters

4. Reporting

  • Balance sheets
  • Income statements
  • Cash flow statements
  • Custom report builder
  • Export to PDF, Excel, CSV

5. User Management

  • Multi-user support
  • Role-based access control
  • Audit trails
  • User preferences

Technical Architecture

Frontend Structure

HTML5

Semantic markup for accessibility and SEO

  • Responsive layouts
  • Forms and validation
  • Progressive Web App support

CSS3

Modern styling and responsive design

  • CSS Grid and Flexbox
  • Custom properties (variables)
  • Mobile-first approach

JavaScript (ES6+)

Interactive functionality

  • Module-based architecture
  • Event-driven programming
  • API integration

User Interface Pages

Navigation Structure

  • / - Dashboard (Home)
  • /accounts - Accounts Management
  • /transactions - Transaction List
  • /transactions/new - New Transaction
  • /reports - Reports Hub
  • /settings - User Settings
  • /admin - Admin Panel

Common UI Components

  • Navigation bar with menu
  • Sidebar navigation
  • Data tables with sorting and filtering
  • Forms with validation
  • Modal dialogs
  • Toast notifications
  • Loading indicators
  • Date pickers
  • Search autocomplete

Data Model

Core Entities

Account

  • id: Unique identifier
  • name: Account name
  • type: Asset, Liability, Equity, Revenue, Expense
  • code: Account code
  • balance: Current balance
  • currency: Currency code
  • parent_id: Parent account (for hierarchy)

Transaction

  • id: Unique identifier
  • date: Transaction date
  • description: Transaction description
  • reference: Reference number
  • entries: Array of journal entries
  • status: Draft, Posted, Void
  • created_by: User who created
  • created_at: Timestamp

Journal Entry

  • account_id: Account reference
  • debit: Debit amount
  • credit: Credit amount
  • memo: Entry memo

User

  • id: Unique identifier
  • username: Login username
  • email: Email address
  • role: Admin, Manager, User, Viewer
  • permissions: Array of permissions
  • preferences: User settings object

API Endpoints (Planned)

Accounts

  • GET /api/accounts - List all accounts
  • GET /api/accounts/:id - Get account details
  • POST /api/accounts - Create new account
  • PUT /api/accounts/:id - Update account
  • DELETE /api/accounts/:id - Delete account

Transactions

  • GET /api/transactions - List transactions
  • GET /api/transactions/:id - Get transaction details
  • POST /api/transactions - Create transaction
  • PUT /api/transactions/:id - Update transaction
  • DELETE /api/transactions/:id - Delete transaction
  • POST /api/transactions/:id/post - Post transaction

Reports

  • GET /api/reports/balance-sheet - Generate balance sheet
  • GET /api/reports/income-statement - Generate income statement
  • GET /api/reports/cash-flow - Generate cash flow statement
  • GET /api/reports/trial-balance - Generate trial balance

Design Principles

  • Simplicity: Clean, intuitive interface
  • Accessibility: WCAG 2.1 AA compliance
  • Performance: Fast load times and responsive interactions
  • Security: Input validation, CSRF protection, secure authentication
  • Modularity: Reusable components and clear separation of concerns
  • Progressive Enhancement: Works without JavaScript, enhanced with it
  • Mobile-First: Responsive design starting from mobile
